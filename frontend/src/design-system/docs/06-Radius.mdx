# Border Radius

Our border radius system provides a consistent approach to creating rounded corners and visual hierarchy through carefully crafted radius values.

## Overview

The border radius system is designed to create visual harmony and improve user experience through consistent corner rounding across all components. Each radius value serves a specific purpose in the interface, from subtle softening to prominent rounded elements.

## Token Definitions

### Border Radius Scale

```typescript
tokens.borderRadius = {
  none: '0px', // Sharp corners
  sm: '2px', // Subtle rounding
  md: '6px', // Standard rounding
  lg: '12px', // Prominent rounding
  xl: '16px', // Large rounding
  '2xl': '24px', // Extra large rounding
  '3xl': '32px', // Very large rounding
  full: '9999px', // Fully rounded (circle/pill)
};
```

## Usage Rules

### 1. **Small Radius (sm - 2px)**

- **Cards**: Subtle softening of sharp corners
- **Buttons**: Minimal rounding for modern feel
- **Inputs**: Slight corner softening
- **Badges**: Small rounded indicators

### 2. **Medium Radius (md - 6px)**

- **Cards**: Standard card corner rounding
- **Buttons**: Primary button styling
- **Modals**: Dialog corner softening
- **Dropdowns**: Menu corner rounding

### 3. **Large Radius (lg - 12px)**

- **Hero sections**: Prominent content areas
- **Feature cards**: Important content containers
- **Navigation**: Main navigation elements
- **Sidebars**: Panel corner rounding

### 4. **Extra Large Radius (xl+ - 16px+)**

- **Hero images**: Large image containers
- **Feature highlights**: Special content areas
- **Decorative elements**: Visual accents
- **Brand elements**: Logo containers

## Best Practices

### 1. **Consistent Application**

```tsx
// ✅ Good - Consistent radius for similar elements
<div style={{ borderRadius: tokens.borderRadius.md }}>
  Card content
</div>

<div style={{ borderRadius: tokens.borderRadius.md }}>
  Another card
</div>

// ❌ Bad - Inconsistent radius values
<div style={{ borderRadius: '6px' }}>
  Card content
</div>

<div style={{ borderRadius: '12px' }}>
  Another card
</div>
```

### 2. **Semantic Usage**

```tsx
// ✅ Good - Semantic radius usage
<button style={{ borderRadius: tokens.borderRadius.md }}>
  Primary Action
</button>

<div style={{ borderRadius: tokens.borderRadius.lg }}>
  Important content area
</div>

// ❌ Bad - Non-semantic usage
<button style={{ borderRadius: tokens.borderRadius['3xl'] }}>
  Over-rounded button
</button>
```

### 3. **Size Appropriateness**

```tsx
// ✅ Good - Radius proportional to element size
<div style={{
  width: '200px',
  height: '100px',
  borderRadius: tokens.borderRadius.lg
}}>
  Large element with large radius
</div>

<div style={{
  width: '60px',
  height: '30px',
  borderRadius: tokens.borderRadius.sm
}}>
  Small element with small radius
</div>

// ❌ Bad - Radius not proportional to size
<div style={{
  width: '60px',
  height: '30px',
  borderRadius: tokens.borderRadius['3xl']
}}>
  Small element with oversized radius
</div>
```

### 4. **Visual Hierarchy**

```tsx
// ✅ Good - Radius creates visual hierarchy
<div style={{ borderRadius: tokens.borderRadius.none }}>
  Base content (sharp corners)
</div>

<div style={{ borderRadius: tokens.borderRadius.md }}>
  Standard content (rounded corners)
</div>

<div style={{ borderRadius: tokens.borderRadius.lg }}>
  Important content (more rounded)
</div>

<div style={{ borderRadius: tokens.borderRadius.xl }}>
  Premium content (highly rounded)
</div>
```

## How-To Guides

### Creating a Card Component

```tsx
import { tokens } from '@/design-system/tokens';

interface CardProps {
  variant?: 'default' | 'elevated' | 'rounded';
  children: React.ReactNode;
}

const Card = ({ variant = 'default', children }: CardProps) => {
  const getRadius = () => {
    switch (variant) {
      case 'default':
        return tokens.borderRadius.md;
      case 'elevated':
        return tokens.borderRadius.lg;
      case 'rounded':
        return tokens.borderRadius.xl;
    }
  };

  return (
    <div
      style={{
        borderRadius: getRadius(),
        padding: tokens.spacing[6],
        backgroundColor: tokens.colors.white,
        border: `1px solid ${tokens.colors.secondary[200]}`,
        transition: 'all 0.2s ease',
      }}
    >
      {children}
    </div>
  );
};
```

### Creating a Button Component

```tsx
interface ButtonProps {
  size?: 'small' | 'medium' | 'large';
  variant?: 'primary' | 'secondary' | 'pill';
  children: React.ReactNode;
}

const Button = ({
  size = 'medium',
  variant = 'primary',
  children,
}: ButtonProps) => {
  const getRadius = () => {
    if (variant === 'pill') return tokens.borderRadius.full;

    switch (size) {
      case 'small':
        return tokens.borderRadius.sm;
      case 'medium':
        return tokens.borderRadius.md;
      case 'large':
        return tokens.borderRadius.lg;
    }
  };

  const getPadding = () => {
    switch (size) {
      case 'small':
        return `${tokens.spacing[2]} ${tokens.spacing[4]}`;
      case 'medium':
        return `${tokens.spacing[3]} ${tokens.spacing[6]}`;
      case 'large':
        return `${tokens.spacing[4]} ${tokens.spacing[8]}`;
    }
  };

  return (
    <button
      style={{
        borderRadius: getRadius(),
        padding: getPadding(),
        backgroundColor:
          variant === 'primary'
            ? tokens.colors.primary[500]
            : tokens.colors.white,
        color:
          variant === 'primary'
            ? tokens.colors.white
            : tokens.colors.secondary[700],
        border:
          variant === 'secondary'
            ? `1px solid ${tokens.colors.secondary[300]}`
            : 'none',
        fontSize: tokens.typography.fontSize.base,
        fontWeight: tokens.typography.fontWeight.medium,
        cursor: 'pointer',
        transition: 'all 0.2s ease',
      }}
    >
      {children}
    </button>
  );
};
```

### Creating an Avatar Component

```tsx
interface AvatarProps {
  size?: 'small' | 'medium' | 'large';
  shape?: 'circle' | 'rounded';
  src?: string;
  alt?: string;
}

const Avatar = ({
  size = 'medium',
  shape = 'circle',
  src,
  alt,
}: AvatarProps) => {
  const getSize = () => {
    switch (size) {
      case 'small':
        return '32px';
      case 'medium':
        return '48px';
      case 'large':
        return '64px';
    }
  };

  const getRadius = () => {
    return shape === 'circle'
      ? tokens.borderRadius.full
      : tokens.borderRadius.lg;
  };

  return (
    <div
      style={{
        width: getSize(),
        height: getSize(),
        borderRadius: getRadius(),
        overflow: 'hidden',
        backgroundColor: tokens.colors.secondary[200],
        display: 'flex',
        alignItems: 'center',
        justifyContent: 'center',
      }}
    >
      {src ? (
        <img
          src={src}
          alt={alt}
          style={{
            width: '100%',
            height: '100%',
            objectFit: 'cover',
          }}
        />
      ) : (
        <div
          style={{
            color: tokens.colors.secondary[600],
            fontSize: tokens.typography.fontSize.sm,
            fontWeight: tokens.typography.fontWeight.medium,
          }}
        >
          {alt?.charAt(0).toUpperCase() || 'U'}
        </div>
      )}
    </div>
  );
};
```

## Accessibility Guidelines

### 1. **Focus Indicators**

- Ensure focus indicators work well with rounded corners
- Test focus rings on elements with different radius values
- Maintain sufficient contrast for focus visibility

### 2. **Touch Targets**

- Ensure rounded buttons maintain adequate touch target size
- Test touch interactions on mobile devices
- Consider minimum touch target guidelines (44px)

### 3. **Screen Reader Support**

- Rounded corners don't affect screen reader functionality
- Maintain semantic HTML structure regardless of visual styling
- Ensure proper ARIA attributes for interactive elements

### 4. **High Contrast Mode**

- Test rounded elements in high contrast mode
- Ensure borders and backgrounds remain visible
- Verify that radius doesn't interfere with contrast requirements

## Testing Checklist

- [ ] All radius values render correctly across browsers
- [ ] Focus indicators work properly with rounded corners
- [ ] Touch targets maintain adequate size with radius
- [ ] Radius values scale appropriately with element size
- [ ] Consistent radius usage across similar components
- [ ] Radius doesn't interfere with accessibility features
