# Dialog Guidelines

## Purpose

Dialogs interrupt the flow for critical tasks or confirmations.

## Usage

```tsx
import {
  Dialog,
  DialogTrigger,
  DialogContent,
  DialogHeader,
  DialogTitle,
  DialogDescription,
  DialogFooter,
  Button,
} from '@/design-system';

export function Example() {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button>Open dialog</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Confirm action</DialogTitle>
          <DialogDescription>This action cannot be undone.</DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button variant="secondary">Cancel</Button>
          <Button variant="destructive">Delete</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

## Dos

- Use for focused tasks and confirmations
- Keep content concise and scannable

## Don’ts

- Don’t use dialogs for long forms—consider dedicated pages/sheets

## Accessibility

- Manage focus on open/close; ensure `aria-labelledby` and `aria-describedby`
