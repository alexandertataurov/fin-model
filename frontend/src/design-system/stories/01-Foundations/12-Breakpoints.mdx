# Breakpoints

Our breakpoint system provides a consistent approach to creating responsive designs that adapt seamlessly across all device sizes and orientations.

## Overview

The breakpoint system is designed to create fluid, responsive experiences that work optimally on every device. It follows a mobile-first approach, ensuring that designs start with mobile considerations and progressively enhance for larger screens.

## Token Definitions

### Breakpoint Values

```typescript
tokens.breakpoints = {
  sm: '640px', // Small devices (phones)
  md: '768px', // Medium devices (tablets)
  lg: '1024px', // Large devices (laptops)
  xl: '1280px', // Extra large devices (desktops)
  '2xl': '1536px', // 2X large devices (large desktops)
};
```

### Responsive Utilities

```typescript
tokens.responsive = {
  mobile: 'max-width: 639px',
  tablet: 'min-width: 640px and max-width: 1023px',
  desktop: 'min-width: 1024px',
  large: 'min-width: 1280px',
  xlarge: 'min-width: 1536px',
};
```

### Container Sizes

```typescript
tokens.containers = {
  sm: '640px',

  xl: '1280px',

```

## Usage Rules

### 1. **Mobile-First Approach**

- **Start with mobile**: Design for smallest screens first
- **Progressive enhancement**: Add features for larger screens
- **Content priority**: Ensure essential content works on mobile

### 2. **Breakpoint Selection**

- **Small (640px)**: Phones and small tablets
- **Medium (768px)**: Tablets and small laptops
- **Large (1024px)**: Laptops and desktops
- **Extra Large (1280px+)**: Large desktops and monitors

### 3. **Responsive Patterns**

- **Stack to grid**: Single column to multi-column layouts
- **Hide/show**: Conditional content based on screen size
- **Scale**: Proportional sizing adjustments
- **Reorganize**: Layout restructuring for different screens

## Best Practices

### 1. **Mobile-First Development**

```tsx
// ✅ Good - Mobile-first approach
<div style={{
  display: 'flex',
  flexDirection: 'column',
  gap: tokens.spacing[4],
  '@media (min-width: 768px)': {
    flexDirection: 'row',
    gap: tokens.spacing[6],
  },
  '@media (min-width: 1024px)': {
    gap: tokens.spacing[8],
  },
}}>
  <div>Content 1</div>
  <div>Content 2</div>
</div>

// ❌ Bad - Desktop-first approach
<div style={{
  display: 'flex',
  flexDirection: 'row',
  gap: tokens.spacing[8],
  '@media (max-width: 1023px)': {
    flexDirection: 'column',
    gap: tokens.spacing[4],
  },
}}>
  <div>Content 1</div>
  <div>Content 2</div>
</div>
```

### 2. **Consistent Breakpoint Usage**

```tsx
// ✅ Good - Consistent breakpoint usage
<div style={{
  maxWidth: '100%',
  padding: tokens.spacing[4],
  '@media (min-width: 768px)': {
    maxWidth: tokens.containers.md,
    padding: tokens.spacing[6],
  },
  '@media (min-width: 1024px)': {
    maxWidth: tokens.containers.lg,
    padding: tokens.spacing[8],
  },
}}>
  Responsive container
</div>

// ❌ Bad - Inconsistent breakpoints
<div style={{
  maxWidth: '100%',
  padding: '16px',
  '@media (min-width: 800px)': {
    maxWidth: '800px',
    padding: '24px',
  },
  '@media (min-width: 1200px)': {
    maxWidth: '1200px',
    padding: '32px',
  },
}}>
  Inconsistent container
</div>
```

### 3. **Content Adaptation**

```tsx
// ✅ Good - Content adaptation for different screens
<div style={{
  fontSize: tokens.typography.fontSize.base,
  lineHeight: tokens.typography.lineHeight.relaxed,
  '@media (min-width: 768px)': {
    fontSize: tokens.typography.fontSize.lg,
    lineHeight: tokens.typography.lineHeight.normal,
  },
  '@media (min-width: 1024px)': {
    fontSize: tokens.typography.fontSize.xl,
    lineHeight: tokens.typography.lineHeight.snug,
  },
}}>
  Responsive typography
</div>

// ❌ Bad - Fixed content sizing
<div style={{
  fontSize: '16px',
  lineHeight: '1.5',
}}>
  Fixed typography
</div>
```

### 4. **Performance Considerations**

```tsx
// ✅ Good - Performance-optimized responsive design
<div style={{
  display: 'grid',
  gridTemplateColumns: '1fr',
  gap: tokens.spacing[4],
  '@media (min-width: 768px)': {
    gridTemplateColumns: 'repeat(auto-fit, minmax(300px, 1fr))',
    gap: tokens.spacing[6],
  },
}}>
  <div>Optimized grid</div>
  <div>Auto-fitting columns</div>
</div>

// ❌ Bad - Performance-heavy responsive design
<div style={{
  display: 'flex',
  flexWrap: 'wrap',
  gap: tokens.spacing[4],
  '@media (min-width: 768px)': {
    flexWrap: 'nowrap',
    gap: tokens.spacing[6],
  },
}}>
  <div>Heavy layout</div>
  <div>Complex calculations</div>
</div>
```

## How-To Guides

### Creating a Responsive Component

```tsx
import { tokens } from '@/design-system/tokens';

interface ResponsiveCardProps {
  title: string;
  content: string;
  variant?: 'default' | 'featured';
}

const ResponsiveCard = ({
  title,
  content,
  variant = 'default',
}: ResponsiveCardProps) => {
  const getResponsiveStyles = () => ({
    padding: tokens.spacing[4],
    borderRadius: tokens.borderRadius.md,
    backgroundColor: tokens.colors.white,
    border: `1px solid ${tokens.colors.secondary[200]}`,
    '@media (min-width: 768px)': {
      padding: tokens.spacing[6],
      borderRadius: tokens.borderRadius.lg,
    },
    '@media (min-width: 1024px)': {
      padding: tokens.spacing[8],
      borderRadius: tokens.borderRadius.xl,
    },
  });

  const getTitleStyles = () => ({
    fontSize: tokens.typography.fontSize.lg,
    fontWeight: tokens.typography.fontWeight.semibold,
    marginBottom: tokens.spacing[3],
    '@media (min-width: 768px)': {
      fontSize: tokens.typography.fontSize.xl,
      marginBottom: tokens.spacing[4],
    },
    '@media (min-width: 1024px)': {
      fontSize: tokens.typography.fontSize['2xl'],
      marginBottom: tokens.spacing[5],
    },
  });

  return (
    <div style={getResponsiveStyles()}>
      <h3 style={getTitleStyles()}>{title}</h3>
      <p
        style={{
          fontSize: tokens.typography.fontSize.base,
          lineHeight: tokens.typography.lineHeight.relaxed,
          color: tokens.colors.secondary[700],
          '@media (min-width: 768px)': {
            fontSize: tokens.typography.fontSize.lg,
            lineHeight: tokens.typography.lineHeight.normal,
          },
        }}
      >
        {content}
      </p>
    </div>
  );
};
```

### Creating a Responsive Grid

```tsx
interface ResponsiveGridProps {
  items: Array<{ id: string; title: string; content: string }>;
  columns?: { mobile: number; tablet: number; desktop: number };
}

const ResponsiveGrid = ({
  items,
  columns = { mobile: 1, tablet: 2, desktop: 3 },
}: ResponsiveGridProps) => {
  const getGridStyles = () => ({
    display: 'grid',
    gap: tokens.spacing[4],
    gridTemplateColumns: `repeat(${columns.mobile}, 1fr)`,
    '@media (min-width: 768px)': {
      gap: tokens.spacing[6],
      gridTemplateColumns: `repeat(${columns.tablet}, 1fr)`,
    },
    '@media (min-width: 1024px)': {
      gap: tokens.spacing[8],
      gridTemplateColumns: `repeat(${columns.desktop}, 1fr)`,
    },
  });

  return (
    <div style={getGridStyles()}>
      {items.map(item => (
        <ResponsiveCard
          key={item.id}
          title={item.title}
          content={item.content}
        />
      ))}
    </div>
  );
};
```

### Creating a Responsive Navigation

```tsx
interface ResponsiveNavigationProps {
  logo: React.ReactNode;
  navItems: React.ReactNode;
  mobileMenu: React.ReactNode;
}

const ResponsiveNavigation = ({
  logo,
  navItems,
  mobileMenu,
}: ResponsiveNavigationProps) => {
  return (
    <nav
      style={{
        display: 'flex',
        alignItems: 'center',
        justifyContent: 'space-between',
        padding: tokens.spacing[4],
        backgroundColor: tokens.colors.white,
        borderBottom: `1px solid ${tokens.colors.secondary[200]}`,
        '@media (min-width: 768px)': {
          padding: tokens.spacing[6],
        },
      }}
    >
      <div>{logo}</div>

      {/* Desktop Navigation */}
      <div
        style={{
          display: 'none',
          '@media (min-width: 768px)': {
            display: 'flex',
            alignItems: 'center',
            gap: tokens.spacing[6],
          },
        }}
      >
        {navItems}
      </div>

      {/* Mobile Menu */}
      <div
        style={{
          display: 'block',
          '@media (min-width: 768px)': {
            display: 'none',
          },
        }}
      >
        {mobileMenu}
      </div>
    </nav>
  );
};
```

## Accessibility Guidelines

### 1. **Touch Targets**

- Ensure minimum 44px touch targets on mobile
- Test touch interactions across all breakpoints
- Maintain adequate spacing between interactive elements

### 2. **Content Scaling**

- Support text scaling up to 200%
- Ensure content remains readable at all sizes
- Test with different zoom levels

### 3. **Navigation**

- Provide accessible navigation on all screen sizes
- Ensure keyboard navigation works across breakpoints
- Test with screen readers on mobile devices

### 4. **Performance**

- Optimize images and assets for different screen sizes
- Use appropriate loading strategies for mobile
- Test performance on lower-end devices

## Testing Checklist

- [ ] Designs work correctly on all target breakpoints
- [ ] Mobile-first approach is implemented
- [ ] Content scales appropriately across screen sizes
- [ ] Touch targets meet accessibility requirements
- [ ] Performance is optimized for mobile devices
- [ ] Navigation works on all screen sizes
- [ ] Content remains readable at all zoom levels
